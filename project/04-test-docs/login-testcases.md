\# 登录接口测试用例 (POST /api/login)



接口信息

\- URL: /api/login

\- 方法：POST

\- 请求体格式: application/JSON

\- 成功条件  ：用户名=admin或user，密码=123456 



测试用例



1. &#x20; 正向用例

TC-01: 管理员正常登录

| 字段 | 内容 |

| 测试点 | 使用正确的管理员账号密码登录 |

| 前置条件 | 无 |

| 输入数据 | `{"username": "admin", "password": "123456"}` |

| 预期状态码 | 200 |

| 预期响应体 | `code: 0`, `message: "登录成功"`, `data.token` 存在 |

| 预期断言 | `res.status === 200` 

`res.body.code === 0` 

`res.body.data.token` 不为空 |



TC-02: 普通用户正常登录

| 字段 | 内容 |

| 测试点 | 使用正确的普通用户账号密码登录 |

| 输入数据 | `{"username": "user", "password": "123456"}` |

| 预期状态码 | 200 |

| 预期响应体 | `code: 0`, `data.user.username === "user"` |



2\.    异常用例 - 认证失败

TC-03: 密码错误

| 字段 | 内容 |

| 测试点 | 正确的用户名 + 错误的密码 |

| 输入数据 | `{"username": "admin", "password": "wrong"}` |

| 预期状态码 | 401 |

| 预期响应体 | `code: 1002`, `message: "密码错误"` |



TC-04: 用户不存在

| 字段 | 内容 |

| 测试点 | 不存在的用户名 |

| 输入数据 | `{"username": "notexist", "password": "123456"}` |

| 预期状态码 | 401 |

| 预期响应体 | `code: 1001`, `message: "用户不存在"` |



TC-05: 用户名正确，密码为空

| 字段 | 内容 |

| 测试点 | 密码为空字符串 |

| 输入数据 | `{"username": "admin", "password": ""}` |

| 预期状态码 | 401（按业务逻辑）或 400（按参数校验） |

| 预期响应体 | `code: 1002` 或 `code: 400` |



3\.   异常用例 - 参数缺失/格式错误



TC-06: 缺少 username 字段

| 字段 | 内容 |

| 测试点 | 请求体中没有 username |

| 输入数据 | `{"password": "123456"}` |

| 预期状态码 | 400 |

| 预期响应体 | 提示缺少必填字段 |



TC-07: 缺少 password 字段

| 字段 | 内容 |

| 测试点 | 请求体中没有 password |

| 输入数据 | `{"username": "admin"}` |

| 预期状态码 | 400 |



&#x20;TC-08: username 为空字符串

| 字段 | 内容 |

| 测试点 | username 传空字符串 |

| 输入数据 | `{"username": "", "password": "123456"}` |

| 预期状态码 | 400 |

| 预期响应体 | 提示用户名不能为空 |



&#x20;TC-09: password 为空字符串

| 字段 | 内容 |

| 测试点 | password 传空字符串 |

| 输入数据 | `{"username": "admin", "password": ""}` |

| 预期状态码 | 400 或 401 |



&#x20;安全用例

TC-10: SQL 注入尝试

| 字段 | 内容 |

| 测试点 | 尝试 SQL 注入攻击 |

| 输入数据 | `{"username": "' OR '1'='1", "password": "123456"}` |

| 预期状态码 | 401（不应返回 200） |

| 预期响应体 | `code: 1001` 或 `code: 1002`，不应返回 token |
